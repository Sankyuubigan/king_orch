## 🤔 Почему в проекте ДВЕ папки "src"?
Программа построена на технологии **Tauri**. Tauri склеивает две разные технологии в одну программу:
1. **Frontend (Папка `src/`)** — Это "Лицо" приложения. Окошки, кнопки, цвета и анимации.
2. **Backend (Папка `src-tauri/`)** — Это "Двигатель". Он работает с жестким диском, загружает ИИ-модели в видеокарту и делает тяжелые вычисления. Внутри нее есть своя папка `src` (код двигателя).

Ниже приведено полное описание всех папок в проекте.

---

## 📁 Корневые папки проекта

- **`agents/`** — Ваша "команда" ИИ-агентов.
  - Файлы `.md` (Markdown) — Инструкции, характеры и правила общения конкретных агентов.
  - Подпапки `workflows/` с файлами `.yaml` — Схемы диалогов (графы). Определяют, какой агент за каким вступает в разговор. Выглядят как визуальные схемы во вкладке "Студия агентов -> Граф".
- **`docs/`** — Документация проекта (включая этот самый файл).
- **`test_cases/`** — Файлы с тестовыми вопросами. Используются во вкладке "Тест агентов ИИ", чтобы автоматически прогнать сотни вопросов через агента и проверить, как он отвечает.
- **`src/`** — Папка Лица (Интерфейса). Подробности ниже.
- **`src-tauri/`** — Папка Двигателя и системных настроек. Подробности ниже.

## 🎨 Лицо приложения: папка `src/`
Здесь находится всё, что отвечает за визуальную часть.

- **`controllers/`** — Управляющие вкладками. Они ловят ваши клики:
  - `chat.ts` — Вкладка "Чат". Поле ввода, кнопки отправки, показ мыслей агентов.
  - `graph.ts` — Редактор графов. Рисование блоков и соединение их стрелочками.
  - `sessions.ts` — Левая панель с историей чатов (переименование, удаление).
  - `settings.ts` — Вкладка "Настройки" (ползунки, скачивание моделей).
  - `agent-test.ts` — Вкладка автоматического тестирования агентов.
- **`ui/`** — "Художники". Рисуют мелкие детали интерфейса:
  - `render.ts` — Рисует облачка сообщений (ваши и от ИИ).
  - `thoughts-block.ts` — Рисует раскрывающийся блок "💭 Мысли".
  - `message-menu.ts` — Выпадающее меню сообщения (⋮) с кнопкой "Удалить".
  - `toast.ts` — Всплывающие цветные уведомления внизу экрана.
  - `confirm.ts` — Всплывающее окно "Вы уверены?".
- **`services/`** — Переводчики. Они берут вашу команду (например, "удали чат") из интерфейса и передают её в двигатель (`src-tauri`).
- **`utils/`** — Вспомогательные инструменты (например, превращение текста агента со звездочками в красивый текст с жирным шрифтом).
- **`store.ts`** — Кратковременная память интерфейса (помнит, какая вкладка открыта).
- **`styles.css`** — Дизайн, цвета кнопок, темная и светлая темы.
- **`main.ts`** — Главный рубильник, запускающий отрисовку интерфейса.

## ⚙️ Двигатель приложения: папка `src-tauri/`
Это "настольная" часть программы. Здесь настраиваются права программы в Windows и лежит сам код движка.

- **`capabilities/`** — Папка с разрешениями для программы. Файл `default.json` внутри нее говорит Windows: "Этой программе разрешено открывать системные окна сохранения файлов и запускать скрипты". Без этого защита компьютера блокировала бы программу.
- **`icons/`** — Картинки и иконки приложения (те, что вы видите на панели задач и на рабочем столе).
- **`mcp_servers/`** — "Руки" искусственного интеллекта. Это скрипты-инструменты, которые агенты могут запускать по своей инициативе:
  - `ddg_search.cjs` — Дает ИИ возможность гуглить (искать в DuckDuckGo).
  - `youtube_mcp.cjs` — Дает ИИ возможность читать субтитры видео с YouTube.
  - `fs_read.cjs` / `fs_write.cjs` — Дают ИИ возможность читать и создавать файлы на вашем диске.
  - `deno_runner.ts` — "Песочница", где ИИ может безопасно запустить написанный им же программный код.
- **`src/`** — Исходный код самого двигателя (написан на языке Rust). Внутри него:
  - **`api/`** — Приемная. Ловит команды от интерфейса (`src/`) и передает их нужным отделам.
  - **`domain/`** — "Мозг". Здесь программа понимает логику агентов (кто кого должен вызвать по схеме, как склеить их ответы).
  - **`infra/`** — "Чернорабочие". Сохраняют чаты на диск, скачивают модели из интернета, загружают ИИ в память компьютера.
- **`tauri.conf.json`** — Главный файл настроек программы (ее название "King Orch", версия, размеры окна при запуске).
- **`Cargo.toml`** — Список стройматериалов (библиотек), нужных для сборки двигателя.
- **`build.rs`** — Инструкция для компилятора, как именно собирать двигатель перед запуском.

## 📄 Отдельные файлы в корне проекта (вне папок)

- **`build.bat` / `build.cjs`** — Скрипты для быстрой локальной сборки программы на вашем компьютере.
- **`release.bat` / `release.cjs`** — Скрипты для создания "чистового" установщика (файла `.exe`, который можно скинуть друзьям).
- **`generate_installer.bat`** — Еще один вариант создания установщика (обычно используется для авто-обновлений).
- **`test.bat`** — Скрипт для проверки, не сломали ли вы двигатель при изменении кода.
- **`models_catalog.json`** — Каталог ИИ-моделей. Из этого файла берется список моделей, когда вы открываете "Настройки -> Скачать новую модель".
- **`package.json`** — Список инструментов, нужных для работы графического интерфейса (Frontend).
- **`index.html`** — Тот самый файл-каркас, на котором рисуется весь графический интерфейс (как веб-страница).
- **`vite.config.ts` / `vitest.config.ts` / `tsconfig.json`** — Технические настройки для сборки и тестирования интерфейса.
